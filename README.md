# 合同修订智能体

一个智能化的合同修订系统，整合了文档提取、法律条款识别、法律条文查询和合同自动修订功能。系统采用模块化设计，支持交互式命令行界面和现代化Web界面，能够智能处理PDF和Word格式的合同文档。

## 系统架构

```
合同文档 (PDF/Word)
    ↓
文档提取模块 (支持中文OCR)
    ↓
法律条款识别模块 (Qwen3-Max)
    ↓
Pinecone向量查询 (支持离线模式)
    ↓
合同修订模块 (Qwen3-Max)
    ↓
修订后的合同 (.txt格式) + 修改建议
    ↓
Web界面 / 命令行界面
```

## 核心功能

### 1. 文档提取模块
- **Word文档提取**: 使用python-docx库提取文本和表格
- **PDF文档提取**: 
  - 自动检测PDF类型（文本版/扫描版）
  - 文本版PDF使用PyPDF2直接提取
  - 扫描版PDF使用PaddleOCR进行OCR识别
  - **支持中文OCR识别**，使用中文模型提高识别准确率

### 2. 法律条款识别模块
- 使用Qwen3-Max大模型分析合同文本
- 自动识别并分类法律相关条款：
  - 责任条款
  - 监管条款
  - 合规条款
  - 争议解决
  - 其他法律条款

### 3. Pinecone向量查询
- 使用SentenceTransformer将法律条款转换为向量
- 在Pinecone索引中查询相似的法律条文
- 返回top-k最相关的法律条文及相似度得分
- **支持离线模式**：网络不佳时自动降级，基于通用法律知识进行修订

### 4. 合同修订模块
- 基于原始合同和相关法律条文
- 使用Qwen3-Max生成修订后的完整合同
- 提供详细的修改建议和修改原因
- 输出格式为.txt文本文件

### 5. 交互式命令行界面
- 友好的用户交互界面
- 自动列出可用的合同文件
- 支持按编号或文件名选择合同
- 实时显示处理进度和状态

### 6. 现代化Web界面 ⭐ 新功能
- 基于Flask的Web服务
- 美观的渐变色设计界面
- 实时合同处理和结果显示
- 结构化的修改建议展示
- 支持修订前后对比
- 响应式设计，支持多设备访问

## 快速开始

### 1. 环境要求
- Python 3.7+
- 稳定的网络连接（可选，支持离线模式）

### 2. 安装依赖

```bash
pip install python-docx pymupdf paddlepaddle paddleocr sentence-transformers pinecone-client openai PyPDF2 pillow numpy
```

### 3. 配置系统

编辑 `config.py` 文件，配置API密钥：

```python
# Qwen3-Max API配置
QWEN_CONFIG = {
    "api_key": "your-api-key",
    "base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "model": "qwen3-max"
}

# Pinecone配置
PINECONE_CONFIG = {
    "api_key": "your-api-key",
    "index_name": "software",
    "top_k": 3
}
```

### 4. 准备合同文件

将需要处理的合同文件（.docx或.pdf格式）放入 `data/contracts` 目录。

### 5. 运行程序

#### 方式一：使用命令行界面

```bash
python main.py
```

程序会：
1. 列出 `data/contracts` 目录中的所有合同文件
2. 提示您选择要处理的合同（输入编号或文件名）
3. 确认后自动处理合同
4. 将修订结果保存到 `data/outputs` 目录，文件名为 `原文件名(revised).txt`

#### 方式二：使用Web界面 ⭐ 新功能

```bash
python src/ui/app.py
```

Web界面特性：
- 🌐 访问地址：http://localhost:5000
- 🎨 现代化渐变色设计界面
- 📝 实时合同处理和结果显示
- 📊 结构化的修改建议展示
- 🔍 支持修订前后对比
- 📱 响应式设计，支持多设备访问

使用步骤：
1. 启动Web服务器：`python src/ui/app.py`
2. 在浏览器中打开：http://localhost:5000
3. 从下拉菜单中选择合同文件
4. 点击"开始处理"按钮
5. 查看修订结果和修改建议

## 工作流程详解

### 步骤1: 文档提取
- 自动识别文件类型（PDF/Word）
- 提取完整的合同文本内容
- 保留文档结构和格式信息
- 支持中文OCR识别，提高中文合同的处理准确率

### 步骤2: 法律条款识别
- 使用Qwen3-Max分析合同文本
- 识别涉及法律规定的条款
- 按照预定义的分类进行标记
- 提取具体的法律条款文本

### 步骤3: 法律条文查询
- 将识别出的法律条款转换为向量
- 在Pinecone中查询相似的法律条文
- 返回最相关的法律条文及相似度
- **离线模式**：网络不佳时跳过此步骤

### 步骤4: 合同修订
- 结合原始合同和相关法律条文
- 使用Qwen3-Max生成修订后的合同
- 提供详细的修改建议和原因
- 输出为.txt格式文本文件

## 输出格式

修订结果包含两个部分：

### 1. 修订后的完整合同
完整的修订后合同文本，可直接使用。

### 2. 修改建议
详细的修改建议列表，包括：
- 修改位置
- 原文内容
- 修改后的内容
- 修改原因

## 注意事项

1. **API密钥**: 确保在 `config.py` 中正确配置所有API密钥
2. **模型下载**: 首次运行会自动下载模型，使用HF-Mirror镜像加速
3. **Pinecone索引**: 确保Pinecone索引中已包含相关的法律条文数据
4. **文件格式**: 目前支持 .docx 和 .pdf 格式的合同文件
5. **网络连接**: 需要稳定的网络连接访问API服务（离线模式下可选）
6. **输出格式**: 输出文件统一为 .txt 格式

## 性能优化建议

1. **模型缓存**: 首次下载模型后会自动缓存，后续使用更快
2. **批量处理**: 可以扩展系统支持批量处理多个合同文件
3. **并行查询**: 可以优化Pinecone查询以提高速度
4. **缓存结果**: 对于相同的法律条款，可以缓存查询结果

## 故障排除

### 问题1: 模型下载失败
- 解决方案: 使用HF-Mirror镜像或手动下载模型
- 配置: 在 `config.py` 中设置 `hf_endpoint`

### 问题2: Pinecone连接失败
- 解决方案: 检查API密钥和网络连接
- 备选方案: 系统会自动切换到离线模式

### 问题3: OCR识别效果差
- 解决方案: 调整DPI参数或使用更高质量的PDF
- 配置: 在 `config.py` 中修改 `text_config` 参数

### 问题4: Qwen3-Max调用失败
- 解决方案: 检查API密钥和配额限制
- 检查: 确保网络连接正常

### 问题5: SentenceTransformer加载超时
- 解决方案: 系统会自动切换到离线模式，无需手动干预
- 说明: 离线模式下系统仍可正常工作
